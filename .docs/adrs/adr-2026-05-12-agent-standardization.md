# ADR-2026-05-12: Standardized Agent Architecture (BaseForgeAgent)

## Context
The RapidForge ecosystem currently lacks a unified way to define, initialize, and delegate tasks to AI Agents. Agents are created ad-hoc, leading to inconsistencies in system prompts, capabilities, and observability. We need a robust, template-derived system for agent lifecycle management.

## Decision
We will implement the **BaseForgeAgent** framework and an **Agent Registry** system.

### 1. BaseForgeAgent & Inheritance
All agents must inherit from a common base class:
- **Properties**: `id`, `category` (INTERNAL | USER_FACING), `role`, `capabilities[]`, `parentTemplate`.
- **System Prompt Compounding**: System prompts are generated by recursively merging parents' prompts (e.g., `BaseForge` -> `SecurityAgent` -> `FirewallAuditor`). This ensures core operational constraints are always present.

### 2. Template System
- Agents are defined via `manifest.yaml` files.
- **Variables**: Support for `{{ PROJECT_NAME }}`, `{{ AUTH_CONTEXT }}`, `{{ @FILE_PATH }}` for dynamic context injection.
- **Logic Gates**: Support for simple conditional logic (`{% if capability == 'shell' %}...{% endif %}`) during prompt synthesis.

### 3. Agent Registry & Lifecycle
- **Registry**: A master manifest (stored in Firestore + local cache) tracks all installed agents.
- **CRUD Tools**:
  - `createAgent()`: Instantiates from a template.
  - `verifyAgent()`: Integrity check of agent definition and required tools.
  - `enableAgent()` / `disableAgent()`: Toggle availability in the ecosystem.
- **Discovery**: An API `listActiveAgents()` is exposed to the POL Layer and the AI Assistant.

### 4. Categorization
- **Internal Agents**: Reserved for POL/Backend tasks (e.g., LogPruner, FailoverManager).
- **User-Facing Agents**: Specialized sub-agents for user delegation (e.g., CodeArchitect, BugHunter).

## Consequences
- Consistent agent behavior and strict adherence to system mandates.
- Improved token efficiency via strategic prompt compounding.
- Unified observability through the EHP bus (all agents emit `agent.activity` events).
